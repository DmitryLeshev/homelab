# Access Planning

## Goal
Согласовать минимально необходимые доступы до начала внедрения.

## Inputs
- Список инженеров и зон ответственности.
- Требуемые операции (bootstrap, day-2 ops, app changes).

## Planning Checklist
1. Определить роли: platform-admin, app-operator, read-only.
2. Зафиксировать где хранится kubeconfig и кто имеет к нему доступ.
3. Определить процесс выдачи и ротации GitHub token.
4. Определить процесс хранения/ротации age private key.
5. Описать break-glass процедуру для инцидентов.

## Acceptance Criteria
- Роли и права документированы.
- Секреты доступа не хранятся в открытом виде в репозитории.
- Есть процедура ротации токенов/ключей.
